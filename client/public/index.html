Prueba de CORS contra la API de Laravel

Este frontend corre en y llama a la API en . Si el CORS está bien configurado, verás los datos; si no, el navegador bloqueará la respuesta con un error de CORS.

1. GET público · /api/articulos

No requiere token. Es la prueba más simple de que CORS funciona.

—
2. POST con credenciales · /api/login

Prueba una petición POST + JSON (pre-flight OPTIONS + Authorization).



—
3. DELETE autenticado · /api/articulos/{articulo}

Usa el Bearer token del login (en memoria). No verifica roles.
